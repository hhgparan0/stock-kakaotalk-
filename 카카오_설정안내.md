# 카톡 자동 브리핑 설정 (컴퓨터 꺼도 발송)

매일 아침 10시쯤 GitHub Actions(무료 클라우드)가 공포지수 + 미국 반도체주를
**내 카카오톡("나에게 보내기")** 으로 쏴줍니다. 컴퓨터/앱이 꺼져 있어도 작동합니다.

설정은 **딱 한 번**만 하면 됩니다. 순서대로 따라오세요.

---

## A. 카카오 디벨로퍼스 앱 만들기

1. https://developers.kakao.com 접속 → 카카오 로그인.
2. 상단 **내 애플리케이션 → 애플리케이션 추가하기** → 앱 이름/사업자명 아무거나 입력 → 저장.
3. 만든 앱 클릭 → 좌측 **앱 키** 메뉴 → **REST API 키** 복사해서 메모.
   👉 이 값이 나중에 시크릿 `KAKAO_REST_API_KEY` 입니다.
4. 좌측 **카카오 로그인** → **활성화 설정 ON**.
5. 같은 화면 아래 **Redirect URI** → `https://localhost` 입력 후 저장.
6. 좌측 **카카오 로그인 → 동의항목** → **"카카오톡 메시지 전송"(talk_message)** 을 **사용**으로 설정(선택 동의로도 충분).
   - "나에게 보내기"는 비즈앱 전환/검수 없이도 본인 계정에 가능합니다.

---

## B. 인증 코드 받기 (브라우저, 1분)

7. 아래 주소에서 `REST_API_KEY` 부분만 3번에서 복사한 키로 바꿔 **브라우저 주소창**에 붙여넣고 이동:

```
https://kauth.kakao.com/oauth/authorize?client_id=REST_API_KEY&redirect_uri=https://localhost&response_type=code&scope=talk_message
```

8. 카카오 로그인 + 동의하기 → `https://localhost/?code=XXXXXXXX...` 로 이동합니다.
   페이지는 안 열려도 됩니다. **주소창의 `code=` 뒤에 오는 값**을 복사하세요.
   ⏱️ 이 코드는 약 10분 안에 다음 단계까지 써야 합니다(만료됨).

---

## C. refresh token 발급 (PowerShell, 한 번만)

9. **PowerShell**을 열고 아래를 실행 (`REST_API_KEY`, `여기에_코드` 두 곳 교체):

```powershell
$body = @{
  grant_type   = "authorization_code"
  client_id    = "REST_API_KEY"
  redirect_uri = "https://localhost"
  code         = "여기에_코드"
}
Invoke-RestMethod -Method Post -Uri "https://kauth.kakao.com/oauth/token" -Body $body
```

10. 출력에서 **`refresh_token`** 값을 복사하세요.
    👉 이 값이 시크릿 `KAKAO_REFRESH_TOKEN` 입니다. (`access_token`은 무시)

> 만약 `KOE320` / `authorization code not found` 오류가 나면 코드가 만료된 것 → B(7~8) 다시.

---

## D. 깃허브 레포 + 시크릿 등록

11. 깃허브에서 **새 레포지토리 생성**(Private 권장). 이 폴더의 파일들을 푸시합니다:
    - `send_briefing.py`
    - `.github/workflows/daily-briefing.yml`
    - (`카카오_설정안내.md`는 올려도 되고 안 올려도 됩니다)

12. 레포 → **Settings → Secrets and variables → Actions → New repository secret** 로 2개 등록:

    | Name | Value |
    |------|-------|
    | `KAKAO_REST_API_KEY`  | A-3에서 복사한 REST API 키 |
    | `KAKAO_REFRESH_TOKEN` | C-10에서 복사한 refresh token |

---

## E. 테스트 & 완료

13. 레포 **Actions** 탭 → 좌측 `daily-stock-briefing` → **Run workflow** 버튼으로 즉시 실행.
14. 잠시 뒤 내 카톡에 브리핑이 오면 성공 🎉
    - 실패하면 Actions 실행 로그를 열어 빨간 단계의 메시지를 확인하세요.

---

## 참고

- **발송 시각**: cron `7 1 * * *` = UTC 01:07 = 한국 **오전 10:07**. 깃허브 무료 cron은
  서버 혼잡 시 몇 분~십몇 분 지연될 수 있습니다(아침 브리핑이라 무방).
- **휴장일**: 주말·미국 공휴일이면 직전 거래일 종가가 나옵니다.
- **토큰 수명**: refresh token은 약 2개월. 만료되면 카톡이 안 오는데, 그때 **B~C 단계만 다시** 해서
  `KAKAO_REFRESH_TOKEN` 시크릿을 새 값으로 바꾸면 됩니다. (자동 갱신을 원하면 말씀하세요.)
- **종목/기준 변경**: `send_briefing.py` 상단의 `STOCKS` 목록, `signal_for()` 기준만 고치면 됩니다.
